# Ship — 커밋부터 PR까지 한번에

코드 작성이 완료된 후 검증 → 리뷰 → 커밋 → 푸시 → PR 생성을 순서대로 진행한다.

사용 예시:
- `/ship`               → 커밋 메시지 자동 생성
- `/ship feat: 로그인 추가` → 제목 직접 지정

---

## STEP 1. 사전 검증 (자동)

아래를 순서대로 실행한다. 실패 시 즉시 중단하고 오류를 출력한다.

```bash
./gradlew lint --quiet
./gradlew testDebugUnitTest --quiet
```

결과 출력:
```
✅ Lint 통과
✅ Unit Test 통과
```

---

## STEP 2. 셀프 리뷰 (자동)

`review.md` 의 절차를 따라 현재 변경사항을 리뷰한다.

---

## STEP 3. 사람 확인 ← 여기서 반드시 멈춘다

변경된 파일 목록과 커밋 메시지 안을 출력하고 반드시 사람의 확인을 기다린다.

```bash
git diff main...HEAD --stat
```

```
## 최종 확인

### 변경 파일
- app/src/.../LoginScreen.kt  (+52 / -3)

### 커밋 메시지 (안)
feat: 로그인 화면 추가

━━━━━━━━━━━━━━━━━━━━━━━━
커밋하고 PR을 올릴까요? (yes/no)
━━━━━━━━━━━━━━━━━━━━━━━━
```

**사람이 yes 라고 답하기 전까지 절대 다음 단계로 넘어가지 않는다.**
문제가 있다고 하면 수정 후 STEP 1부터 다시 시작한다.

---

## STEP 4. 커밋 (승인 후 자동)

`commit.md` 의 절차를 따른다. $ARGUMENTS 가 있으면 커밋 메시지 제목으로 사용한다.

---

## STEP 5. 푸시 (자동)

`push.md` 의 절차를 따른다.

---

## STEP 6. PR 생성 (자동)

`pr.md` 의 절차를 따른다. 단, 사전 검증(lint/test)은 STEP 1에서 이미 완료했으므로 건너뛴다.

완료 후 PR URL을 출력한다.
