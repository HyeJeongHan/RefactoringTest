# 셀프 코드 리뷰

PR 올리기 전 현재 브랜치의 변경사항을 리뷰한다.

## 1. 변경사항 파악

```bash
git diff main...HEAD
git log main...HEAD --oneline
```

## 2. 리뷰 기준

`.claude/rules/review-criteria.md` 의 항목을 **빠짐없이** 체크한다.

## 3. 결과 출력

```
## 셀프 리뷰 결과

### ✅ 통과 항목
- ...

### ❌ 문제 발견
- [파일명:라인] 문제 설명
  → 개선 제안

### 💡 추가 제안 (선택)
- ...
```

문제가 없으면 "리뷰 통과 — PR 올려도 좋습니다!" 를 출력한다.
문제가 있으면 수정 후 재리뷰를 권장한다.
